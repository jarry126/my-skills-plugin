# 周报风格库

SKILL.md 正文里只放基础结构。需要套用具体风格时，读本文件对应章节。

## team（团队内部，默认）
- 语气：平实、口语化，像同事间同步。
- 重点：进展和协作，可带少量背景。
- 三段式：本周完成 / 下周计划 / 问题与支持。

示例：
```
## 一、本周完成
- 登录页改版上线，新版本注册转化率 +12%
- 修复 3 个线上 bug（详见 Jira PROJ-101/102/103）
```

## boss（向上汇报）
- 语气：正式、结论先行，突出价值与风险。
- 重点：成果对目标的贡献、需要决策/资源的事项。
- 结构：核心成果 / 关键指标 / 风险与需求 / 下周重点。

示例：
```
## 核心成果
- 完成登录页改版，注册转化率提升 12%，预计季度多带来约 800 注册用户。

## 风险与需求
- 支付模块依赖外部接口，需在周三前确认对接方排期。
```

## brief（极简）
- 语气：最短，每段不超过 3 条。
- 只保留：做了什么 / 下周做什么。去掉一切修饰。

示例：
```
本周：登录改版上线、修3个bug
下周：开发支付模块
```
