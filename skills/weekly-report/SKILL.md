# 周报生成

把用户提供的零散工作内容，整理成结构统一、可直接发送的周报。

## 何时使用
- 用户说"写周报""周总结""整理这周的工作""weekly report"
- 用户给了一堆零散的本周事项，想要规整成正式格式

## 收集信息
开始前确认以下信息，缺哪个就问用户（一次只问缺的）：
1. 本周完成的事项（必需）
2. 下周计划（必需）
3. 遇到的问题 / 需要的支持（可选）
4. 风格偏好：team（团队内部）/ boss（向上汇报）/ brief（极简）。默认 team。

## 步骤
1. 若用户没给本周日期范围，运行脚本计算：
   `python3 scripts/date_range.py`
   它会输出本周一到周日的日期，把结果填进标题。
2. 根据用户选择的风格，查阅 `references/styles.md` 对应章节，套用该风格的语气和结构。
3. 按下面的基础结构生成周报。

## 输出格式（基础结构）
```
# 周报（{起始日期} ~ {结束日期}）

## 一、本周完成
- {事项}：{一句话结果/数据}

## 二、下周计划
- {计划事项}

## 三、问题与支持
- {问题}（如无则写"无"）
```

## 写作要求
- 每条**先写结果、再写细节**，能量化就量化（如"完成 X，转化率提升 12%"）。
- 用动词开头，简洁，一条一行，不写流水账。
- 不同风格的具体差异见 `references/styles.md`。

## 注意
- 不要编造数据或成果；用户没给的就不写，或标注"待补充"。
- 完成后问用户是否需要导出为 Word / 调整风格。
